# 宋肆小筑：零经验游戏开发起步原型

这是一个模仿《暖暖温泉乡》经营循环的宋代食肆小游戏原型。它不是完整游戏，而是第 1 课用来理解“可玩闭环”的版本。

## 怎么打开

直接双击 `index.html`，或在浏览器中打开：

```text
C:\Users\linrj\Documents\休闲\song_eatery_game\index.html
```

## 发布到 GitHub Pages

这个原型是纯静态网页，不需要服务器代码。上传到 GitHub 后，可以这样发布：

1. 在 GitHub 新建一个仓库，例如 `song-eatery-game`。
2. 把本文件夹里的 `index.html`、`styles.css`、`game.js`、`README.md` 上传到仓库根目录。
3. 打开仓库的 `Settings` -> `Pages`。
4. 在 `Build and deployment` 里选择 `Deploy from a branch`。
5. Branch 选择 `main`，Folder 选择 `/root`，然后保存。
6. 等 GitHub Pages 构建完成后，访问页面给出的 `https://用户名.github.io/song-eatery-game/` 链接。

## 现在能玩什么

- 在格子地图上铺木地、摆食案、添灶台、放花窗。
- 点击“开市迎客”，客人会按食案数量入座点菜。
- 灶台不足时，客人等待变久，满意度会下降。
- 研制新菜可以提高单客收入和名声。
- 名声越高，每天可来的客人越多。
- 食案靠近花窗会提高满意、客单价和名声。
- 食案靠近灶台会被烟火影响，满意度下降。
- 可以雇跑堂提高接待能力，但每天要发薪水。
- 不同客群有不同偏好，名声提高后会解锁新客群。
- 客群图鉴会记录每类客人的到访次数和累计贡献。
- 浏览器会自动保存经营进度，也可以手动重开一局。
- 月份节庆会轮换出现，影响客流、客单价、满意和名声。
- 多个设施摆成特定组合时，会触发相性加成。
- 菜品分成羹汤、面点、酒水、节令菜，不同菜系会给经营带来不同加成。
- 跑堂可以升级，效率提高的同时日薪也会增加。

## 你要先学的 4 件事

1. HTML：页面有哪些按钮、区域、文字。
2. CSS：游戏看起来像什么。
3. JavaScript 状态：钱、名声、格子、客人都存在 `state` 里。
4. 游戏循环：建造设施 -> 开市迎客 -> 结算收益 -> 解锁新菜 -> 继续扩建。

## 第 2 课：布局策略

开罗式经营游戏最重要的乐趣之一，是“同样的设施，摆法不同，结果不同”。现在原型里已经有两条简单规则：

- 花窗贴着食案：客人觉得雅致，满意和收入会上升。
- 灶台贴着食案：出菜方便但烟火扰客，满意会下降。

你可以试着把两张食案摆成不同位置，然后点“开市迎客”，观察右侧“布局效果”和掌柜札记里的结算文字。

## 第 3 课：员工和固定成本

经营游戏不能只让收入上涨，还要让玩家承担扩张成本。现在原型里加入了“跑堂”：

- 雇跑堂要先付一笔钱。
- 每个跑堂每天要发薪水。
- 跑堂会提高接待能力，让更多客人不至于久等。

这对应了开罗游戏里常见的“雇人 -> 效率提升 -> 支出增加 -> 需要更高收入覆盖成本”的循环。

## 第 4 课：客群偏好

经营游戏的中期目标，通常来自“我想吸引谁”。现在原型里加入了客群：

- 行商：一开始就会来，客单价略高。
- 书生：一开始就会来，喜欢临窗雅座。
- 茶博士：名声 8 解锁，耐心更好。
- 瓦舍伶人：名声 16 解锁，出手阔但催菜急。
- 船客：名声 24 解锁，不喜欢灶烟贴席。

这会让设施布局、员工和名声互相勾连：不是所有客人都只看“有没有桌子”，他们会被不同条件影响。

## 第 5 课：客群图鉴

经营游戏需要“短期收益”和“长期收藏”两条线。现在原型里加入了客群图鉴：

- 每类客人会记录到访次数。
- 每类客人会记录累计贡献的铜钱。
- 未解锁客群会先显示解锁条件。

这样玩家就会有新的目标：不只是今天赚多少钱，还会想把所有客群都吸引来，并看哪类客人最值钱。

## 第 6 课：本地存档

经营游戏需要保存进度，否则玩家很难长期投入。现在原型使用浏览器的 `localStorage` 做本地存档：

- 建造、拆除、研制新菜、雇跑堂、开市结算后会自动保存。
- 刷新页面后会恢复铜钱、名声、满意、员工、布局和客群图鉴。
- 点击“重开一局”会清空旧存档并重新开始。

这是最适合网页原型的轻量存档方式，等以后接后端或账号系统时，再换成云端存档。

## 第 7 课：月份节庆

开罗式经营游戏常用“时间变化”制造期待感：玩家不只是重复同一天，而是在等下一个活动、下一个旺季、下一个需求变化。

现在原型里加入了 4 个轮换节庆，每个节庆持续 3 天：

- 正月：上元灯会，客流增加，名声更容易上涨。
- 三月：寒食踏青，满意度更容易提高。
- 五月：端午竞渡，船客会提前出现，客单价提高。
- 八月：中秋赏月，客流和满意都会提高。

这一步让游戏开始有“经营节奏”：平时可以攒钱修布局，旺季则要提前准备更多食案、灶台和跑堂。

## 第 8 课：设施相性

开罗游戏的布局乐趣，不只是“这个设施本身有什么用”，还在于“它和旁边设施摆在一起会不会更强”。这会让玩家自然开始反复调整店面。

现在原型里加入了 4 个相性：

- 双灶成房：两口灶台相邻，出菜能力提高。
- 花窗成景：花窗达到 2 扇，满意和名声提高。
- 雅座成区：临窗食案达到 2 张，客单价和名声提高。
- 堂口开阔：食案达到 3 张，额外增加客流。

这一步很重要：玩家开始不是为了“把空地填满”而建造，而是为了凑出更好的经营组合。

## 第 9 课：菜系研究线

经营游戏里的“研究”最好不要只是数字变大。更有趣的做法是让玩家感觉自己在选择方向：要做稳定满意的羹汤，还是提高客单的面点和酒水，还是趁节庆打名声的节令菜。

现在原型里把菜品拆成 4 条菜线：

- 羹汤：提高满意，让客人评价更稳。
- 面点：提高客单价，适合补现金流。
- 酒水：行商和瓦舍伶人更愿意多付钱。
- 节令菜：节庆期间更容易涨名声。

研制新菜时，按钮会显示下一道菜属于哪条菜线；菜牌里也会显示每条菜线当前解锁进度。这样玩家会开始形成“我这局先走哪条经营路线”的想法。

## 第 10 课：员工等级

经营游戏里的员工不应该只是“多雇几个”。如果员工能升级，玩家就会面对更有趣的取舍：是花钱扩人数，还是培养现有人手；是追求更高效率，还是担心每天薪水变高。

现在原型里跑堂有了等级：

- 雇跑堂：增加人手数量。
- 升跑堂：提高每个跑堂的接待能力。
- 等级越高：客人越不容易久等，但每日薪水也会增加。
- 最高 4 级：避免早期数值失控。

这一步对应开罗游戏常见的“员工养成”循环：投入培养 -> 效率提高 -> 固定成本增加 -> 需要更好的布局和菜品来覆盖成本。
